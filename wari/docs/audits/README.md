# Wari — Audit Reports

Dated audit documents, one per phase milestone (plus ad-hoc when
triggered). Each report contains: scope, findings, severity,
remediation plan, and sign-off.

Populated as milestones close:

  - `phase-0.md` — at milestone m0
  - `phase-1.md` — at milestone m1
  - `phase-2.md` — at milestone m2
  - `phase-3.md` — at milestone m3 (external security firm)
  - `phase-4.md` — pre-tapeout (formal verification coverage)

Ad-hoc audits (post-CVE, post-incident) use the naming
`YYYY-MM-DD-<subject>.md`.
